# Security policy

## Supported versions

Security updates are applied to the **`main`** branch as needed. There is no separate long-term support branch for this early-stage project.

## Reporting a vulnerability

**Please do not open a public GitHub issue** for undisclosed security problems (e.g. remote code execution, credential leaks in tooling, or supply-chain issues in build scripts).

Instead:

1. Open a **private security advisory** on GitHub (**Security → Advisories → Report a vulnerability**), or  
2. Contact the repository owner through their GitHub profile if advisories are unavailable.

Include:

- A short description of the issue and its impact  
- Steps to reproduce (or proof-of-concept) if you can share them safely  
- Affected components (Godot version, scripts, CI, etc.)

We will acknowledge receipt when we can and coordinate a fix and disclosure timeline.

## Scope

This project is a **Godot game** and supporting tools. Reports about upstream Godot Engine vulnerabilities should generally go to the [Godot project](https://github.com/godotengine/godot/security) unless they only affect our scripts or data in this repo.
